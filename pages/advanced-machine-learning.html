CodeT5 Natural Language to Python Code Generation

CodeT5-base를 CoNaLa 자연어-코드 데이터셋으로 fine-tuning해 자연어 지시문을 Python 코드로 변환하는 sequence-to-sequence 모델을 구현하고 실험했습니다.

CourseAdvanced Machine Learning
ModelCodeT5-base
DatasetCoNaLa
MetricBLEU 14.48

Project Overview

프로젝트의 목표는 사용자가 자연어로 작성한 프로그래밍 의도를 Python 코드로 변환하는 모델을 만드는 것이었습니다. Salesforce CodeT5-base를 기반으로 CoNaLa의 자연어 질문과 Python 코드 쌍을 학습시키고, 생성된 코드가 참조 코드와 얼마나 유사한지 BLEU로 평가했습니다.

Natural Language to Code Seq2Seq Transformer Fine-tuning BLEU Evaluation

Experiment Direction

단순히 모델을 한 번 학습시키는 데서 끝내지 않고, 모델 크기, 학습 길이, 입력·출력 토큰 길이, 생성 전략을 조정하면서 손실과 생성 예시가 어떻게 변하는지 확인했습니다. 실험의 메시지는 명확했습니다. 코드 생성은 모델 크기만 키운다고 해결되는 문제가 아니라, 데이터 전처리와 decoding strategy가 함께 맞아야 자연어 의도가 코드 구조로 안정적으로 옮겨진다는 점입니다.

01 Baseline Review

CodeT5-small과 짧은 학습 설정으로는 간단한 패턴은 잡지만 질문 의도를 충분히 반영하기 어려운 한계를 확인

02 Model Scale-up

CodeT5-base로 전환해 파라미터 규모를 60M 수준에서 약 220M으로 확장

03 Training Setup

epoch 5, learning rate 5e-5, AdamW, weight decay 0.01, warmup 10%로 학습 안정화

04 Generation Tuning

beam size 8, temperature 0.7, top-p 0.9, no-repeat 2-gram으로 반복을 줄이고 후보 탐색 폭을 조정

What I Learned

  • 자연어-코드 변환은 일반적인 텍스트 생성보다 정답 공간이 좁고, 문법·API 선택·의도 반영을 동시에 만족해야 합니다.
  • loss 감소는 학습이 진행되고 있음을 보여주지만, 실제 코드 품질은 BLEU와 생성 예시를 함께 봐야 판단할 수 있습니다.
  • CodeT5-base로 확장하고 sequence length를 256으로 늘리면서 더 긴 질문과 코드 패턴을 다룰 수 있었습니다.
  • beam search, temperature, top-p, no-repeat n-gram은 생성 결과의 안정성과 다양성 사이의 균형을 조정하는 실험 포인트였습니다.
  • 최종 BLEU 14.48은 간단한 Python idiom 생성 가능성을 보여주지만, 복잡한 알고리즘 문제나 실행 정확성까지 보장하려면 더 큰 데이터셋과 execution-based evaluation이 필요합니다.
