BUIDL Asia Story Track

AI 학습 데이터로 사용되는 창작물의 IP와 기여도를 추적하고, 생성 결과에 따른 보상을 원작자에게 분배하는 블록체인 기반 IP 보상 플랫폼 StoryTuning을 개발해 BUIDL Asia Story Track에서 최우수상을 수상했습니다.

Project Overview

AI 이미지 모델은 수많은 창작물을 학습하지만, 어떤 데이터가 학습에 쓰였는지와 그 기여가 누구에게 보상되는지는 불투명합니다. StoryTuning은 창작자가 이미지를 IP로 등록하고, 그 이미지가 모델 fine-tuning과 생성 결과에 어떻게 기여했는지 기록해 보상 분배까지 연결하는 구조를 제안했습니다.

AI Dataset IP Story Protocol Fine-tuning Royalty Distribution

Problem Definition

  • 생성형 AI 모델은 원작자의 이미지와 스타일을 학습하지만, 학습 데이터의 출처와 사용 내역이 공개되지 않는 경우가 많음
  • 창작물 기반 모델이 수익을 만들더라도 원작자에게 직접적인 보상 흐름이 돌아가기 어려움
  • GPT, Claude 같은 closed API는 실제 학습 데이터와 fine-tuning 구조가 불투명해 IP 보상 실험에 적합하지 않음
  • 창작자 등록, 학습 로그, 생성 결과, 로열티 정산을 하나의 흐름으로 묶는 서비스 구조가 필요함

Core Idea

기본 AI 파이프라인은 dataset, training, model의 순서로 동작합니다. StoryTuning은 이 흐름에서 데이터셋에 포함된 이미지의 IP를 먼저 등록하고, fine-tuning에 사용된 이미지와 생성 결과의 유사도를 기반으로 기여도를 계산해 보상 분배 기준을 만드는 방식으로 설계했습니다.

IP Registration 창작자가 업로드한 이미지를 Story Protocol 기반 IP asset으로 등록
Training Log fine-tuning에 사용된 이미지, metadata, 학습 로그를 기록
Contribution 생성 이미지와 학습 이미지의 연관도를 바탕으로 기여도 산정
Royalty 스마트 컨트랙트로 원작자에게 보상을 분배하는 흐름 구성

Service Flow

Creator

  1. 프론트엔드에서 여러 장의 원본 이미지 업로드
  2. 이미지 metadata와 IP 정보를 Story Protocol에 기록
  3. 학습 데이터셋으로 선택된 이미지와 사용 로그 확인
  4. 생성 결과 기여도에 따라 로열티 보상 수령

User

  1. fine-tuning된 AI 모델을 선택하고 프롬프트 입력
  2. 생성 이미지 결과 확인
  3. 생성에 기여한 학습 이미지와 보상 흐름 확인
  4. 생성 이미지 자체도 온체인 기록 대상으로 연결

Architecture

Frontend

  • 메인 화면, 이미지 업로드, 모델 선택, 이미지 생성 UI 구성
  • 창작자와 사용자가 IP 등록 및 생성 흐름을 확인하는 접점

Backend / DB

  • AI 학습/생성 API 연결과 request 처리
  • 이미지 저장, metadata 저장, 학습 로그와 생성 기록 관리

AI

  • Stable Diffusion, Hugging Face, 자체 dataset 기반 fine-tuning 흐름 검토
  • closed API 대신 직접 학습 파이프라인을 구성하는 방향으로 설계

Blockchain

  • Story SDK를 통한 IP 등록, 학습 로그, 생성 결과 metadata 기록
  • royalty distribution contract로 원작자 보상 분배 구조 연결

Key Features

  • 창작자는 자신의 콘텐츠를 IP로 등록하고 fine-tuned model의 권리 구조를 가질 수 있음
  • 사용자는 특정 모델을 사용하기 위해 license token 또는 보상 지불 흐름을 거침
  • 생성 결과는 학습 데이터 기여도와 연결되어 원작자 보상 근거로 사용됨

Project Direction

단순히 이미지를 생성하는 서비스가 아니라, AI 모델이 어떤 데이터로 학습되었고 누가 그 데이터의 권리를 갖는지 보여주는 구조에 초점을 맞췄습니다. 창작자에게는 IP 등록과 수익화 경로를, 사용자에게는 provenance가 남는 생성 경험을 제공하는 것이 목표였습니다.

My Work

  • AI 학습 데이터 IP 보상 문제를 프로젝트 주제로 정리하고 서비스 흐름 구체화
  • 창작자 업로드, IP 등록, 학습 로그, 사용자 생성, 로열티 분배까지 이어지는 end-to-end flow 설계
  • frontend, backend, AI, blockchain 역할을 분리해 전체 아키텍처 구성
  • BUIDL Asia Story Track 발표 자료와 수상용 narrative 정리
