BattleContract 1:1 Social Polymarket

Project Overview

BattleContract는 말로 끝나는 논쟁을 책임 있는 1:1 배틀로 바꾸는 Web3 서비스입니다. 사용자는 자신의 주장에 토큰을 걸고, 반대 의견을 가진 사용자가 같은 금액으로 참여하면 배틀이 성사됩니다. 이후 중립 위원회가 양쪽 의견을 판단하고, 결과에 따라 스마트 컨트랙트가 승자에게 보상을 자동 분배합니다.

1:1 Opinion Battle Token Staking Committee Voting Smart Contract Settlement

Problem Definition

  • 일상 속 논쟁은 많지만 대부분 명확한 결론 없이 의견으로만 끝남
  • 주장에 책임이 없기 때문에 참여자의 태도와 몰입도가 쉽게 낮아짐
  • 기존 예측 시장은 외부 사실 기반 결과에 강하지만 주관적 의견 대결에는 적합하지 않음
  • 논쟁을 콘텐츠로 만들면서도 공정한 판단과 보상 정산을 함께 처리하는 구조가 부족함

Service Flow

User A

  1. 논쟁 주제 생성
  2. 자신의 의견과 배팅 금액 설정
  3. 위원 수를 설정하고 배틀을 공개

User B

  1. 반대 의견으로 배틀 참여
  2. 동일한 금액을 stake
  3. 1:1 head-to-head 배틀 성사

Committee

  1. 퀴즈 통과로 위원 자격 획득
  2. 양측 의견을 검토하고 중립 투표
  3. 투표 결과로 승자 결정

Contract

  1. 배팅 금액 자동 예치
  2. 투표 결과 확정
  3. 승자에게 보상 자동 전송

Core Mechanics

  • 주제는 플랫폼이 아니라 사용자가 직접 생성해 밈, 스포츠, 제품 선호, 사회적 논쟁까지 확장 가능
  • 배팅 금액을 걸어 의견에 책임을 부여하고, 관객은 단순 투표자가 아닌 판단 주체로 참여
  • 위원회는 퀴즈를 통과한 사용자로 구성해 무작위 투표보다 높은 신뢰도를 확보
  • 결과는 말싸움이 아니라 기록 가능한 Web3 프로세스로 정리되어 공유 가능한 콘텐츠가 됨

Web3 Trust System

지갑 기반 로그인, 스마트 컨트랙트 예치, 위원회 투표, 자동 정산을 결합해 중앙화된 심판 없이도 배틀 결과를 처리하는 구조를 제안했습니다. 플랫폼이 임의로 승자를 정하지 않고, 투표 결과와 보상 분배 흐름을 블록체인에 남기는 것이 핵심입니다.

Wallet Login 사용자는 지갑으로 입장하고 배틀 참여 권한을 연결합니다.
Escrow 양측 stake는 스마트 컨트랙트에 예치되어 결과 확정 전까지 보관됩니다.
Settlement 위원회 투표 결과에 따라 승자에게 토큰이 자동 전송됩니다.

Expansion Direction

  • 위원회를 DAO 형태로 확장해 커뮤니티가 판단 시스템을 운영하는 구조로 발전 가능
  • NFT 멤버십으로 private battle, community-only battle, premium judging을 구성 가능
  • 배틀 결과를 밈이나 소셜 콘텐츠로 전환해 확산 가능한 debate network로 확장 가능
  • 단발성 배틀을 넘어 연속 배틀, 랭킹, reputation, 시즌제 보상 구조로 게임화 가능

My Work

  • 문제 정의와 서비스 내러티브를 정리하고 발표 흐름 구성
  • 1:1 배틀 생성, 반대 의견 참여, 위원회 투표, 보상 정산 흐름을 서비스 구조로 구체화
  • 기존 Polymarket과의 차별점을 1:1 구조, 주관적 주제, 커뮤니티 판단 중심으로 정리
  • 해커톤 환경에서 프로토타입 구현과 발표 자료 구성을 병행
