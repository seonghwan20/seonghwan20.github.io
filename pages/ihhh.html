Activity Overview

IHHH는 홍익대학교 정보보안학회 활동으로, 시스템 해킹과 리버스 엔지니어링 기법을 학습하고 보안 취약점을 직접 분석하는 방식으로 진행했습니다. Kali Linux 환경을 구축해 실습을 진행했고, Dreamhack wargame 문제를 풀며 학습한 개념을 반복적으로 적용했습니다.

Weekly Study Linux Binary Pwnable CTF Practice

Study Routine

  • 주 2회 정기 스터디를 통해 시스템 해킹 이론과 실습 문제를 병행
  • Kali Linux 기반 실습 환경에서 gdb, pwndbg, pwntools를 활용해 바이너리 동작 분석
  • Dreamhack wargame을 통해 취약점 트리거, 메모리 상태 확인, exploit script 작성 흐름을 반복 연습
  • 2024 핵테온 세종 국제 대학생 사이버보안 경진대회 CTF 예선에 참가해 문제 해결 경험 확장

System Hacking Focus

Binary Analysis

  • ELF 실행 구조, entry point, section, shared object 매핑 이해
  • gdb와 pwndbg로 register, stack, disassembly, backtrace 상태 추적
  • checksec와 vmmap을 활용해 보호 기법과 메모리 레이아웃 확인

Memory Vulnerability

  • stack buffer overflow로 데이터 변조, 정보 유출, 실행 흐름 조작 원리 학습
  • return address overwrite, stack canary leak, out-of-bounds, format string bug 분석
  • NX, ASLR, PIE, RELRO가 exploit 전략에 미치는 영향 정리

Exploit Practice

  • open-read-write shellcode와 execve shellcode 작성 및 opcode 추출
  • pwntools로 process, remote, send/recv, packing/unpacking 자동화
  • ROP chain, ret2libc, GOT overwrite 등 보호 기법 우회 흐름 실습

Reverse Engineering Scope

리버스 엔지니어링은 시스템 해킹 실습을 보조하는 분석 역량으로 학습했습니다. 컴파일 과정, ELF와 PE 같은 실행 파일 형식, disassemble과 decompile의 차이를 정리하고, 정적 분석과 동적 분석을 상황에 따라 선택하는 기준을 익혔습니다.

  • source code가 binary로 변환되는 preprocessing, compile, assemble, link 단계 이해
  • disassembler와 decompiler를 활용한 프로그램 구조 파악
  • static analysis와 dynamic analysis의 장단점, 난독화와 anti-debugging 제약 학습

What I Practiced

Debugging 중단점 설정, 레지스터와 스택 관찰, 함수 호출 흐름 추적으로 취약점 발생 지점 확인
Exploit Flow 취약점 조건 확인, payload 구성, 주소 계산, exploit script 작성까지의 흐름 반복
CTF Readiness Dreamhack wargame과 CTF 예선 참여를 통해 제한 시간 안에서 문제를 분석하는 방식 경험
