SAP CODE 1기

SAP ERP 실무 개발과 업무 프로세스를 프로젝트 중심으로 학습하고, ERP 구축 프로젝트의 PM 및 CO 담당 경험으로 연결한 활동입니다.

Overview

SAP ERP 실무 개발, 업무 프로세스, 프로젝트 멘토링을 중심으로 진행된 과정입니다. FI, MM, SD, PP, CO 등 ERP 모듈의 연결 구조를 이해하고, ABAP과 Fiori 기반 산출물을 프로젝트에 적용했습니다.

Activity Focus

  • SAP ERP 주요 모듈과 업무 흐름 이해
  • ABAP Program, CDS View, Fiori App 산출물 작성
  • 프로젝트 멘토링 기반 요구사항 정리와 일정 관리
  • ERP 구축 프로젝트에서 PM 및 CO 담당 역할 수행

Mentoring Activity

ABAP 기반 데이터 처리, RAP/OData 서비스 노출, Fiori 화면 구성 흐름을 주차별로 정리했습니다. 단순 문법 정리보다 실제 프로젝트에서 어떤 레이어가 어떤 역할을 맡는지 연결해 이해하는 데 초점을 두었습니다.

Week 1 ADBC · AMDP · RAP/OData

Open SQL과 Native SQL 접근 차이를 비교하고, ADBC를 통한 직접 DB 접근과 AMDP의 Code-to-Data 처리 방식을 정리했습니다. CDS, RAP, OData, Fiori로 이어지는 S/4HANA 개발 구조도 함께 정리했습니다.

Week 2 Fiori Master-Detail

Binding Context, OData Key Path, bindElement, Association, $expand를 중심으로 리스트에서 상세 화면으로 데이터 기준점이 이동하는 구조를 정리했습니다.

Week 3 Gateway · OData Request Flow

Fiori 화면 요청이 OData URI를 통해 Gateway로 전달되고, DPC_EXT의 CRUD 메서드와 HANA 데이터 조회로 이어지는 흐름을 정리했습니다.

Week 3+ Metadata Extension · BDEF · EML

Metadata Layer 우선순위, UI Annotation, Behavior Definition, Validation, Determination, Action, EML 기반 READ/MODIFY ENTITIES 흐름을 정리했습니다.

Mentoring Notes

  • DB 접근, 서비스 노출, 화면 구성 레이어를 분리해 SAP 개발 아키텍처를 정리
  • Fiori와 Gateway 사이의 URI, EntitySet, CRUD 매핑 흐름을 실습 중심으로 정리
  • RAP 기반 앱 구현에 필요한 CDS, MDE, BDEF, Service Binding 순서를 개발 프로세스로 정리
  • 프로젝트 화면 개발과 CO 분석 화면 구현에 활용할 수 있는 OData/Fiori 개념을 축적

Connected Project

이 과정에서 학습한 ERP 프로세스와 개발 산출물은 SAP CODE ERP Implementation Project의 PM, CO 프로세스 설계, 모듈 연계 흐름 정리로 이어졌습니다.
