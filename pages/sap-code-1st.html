ABAP Development

Data Structure

Internal Table과 Work Area를 기준으로 대량 데이터를 처리하고, Field Symbol로 동적 참조와 반복 처리 비용을 줄이는 방식을 학습했습니다.

Report Program

INITIALIZATION, AT SELECTION-SCREEN, START-OF-SELECTION 흐름을 구분하고 선택 조건 검증과 실행 로직을 분리했습니다.

Screen Logic

Selection Screen, PBO, PAI, Module Pool, Tabstrip, Subscreen 구조를 통해 SAP GUI 화면 이벤트 흐름을 정리했습니다.

ALV Control

Field Catalog, Layout, Event Handler, Local Class를 사용해 리스트 조회 화면과 사용자 이벤트를 구성했습니다.

Object · Function

Function Module, Exception, Class/Method, Static/Instance 호출, Inheritance 접근 범위를 실습 단위로 정리했습니다.

External Data Handling

BDC, ADBC, Native SQL 접근을 다루며 SAP 표준 화면 자동 입력과 DB 직접 조회의 역할 차이를 확인했습니다.

Fiori · OData · RAP

01 UI5 App

index.html, Component.js, manifest.json, View XML, Controller 구조

02 Model Binding

JSONModel, ODataModel V2, named model, expression binding, binding context

03 Gateway

Metadata, EntityType, EntitySet, URI, $filter, $expand, Function Import

04 DPC_EXT

GET_ENTITYSET, CREATE_ENTITY, UPDATE_ENTITY, DELETE_ENTITY 구현

05 RAP

CDS, MDE, BDEF, Behavior Implementation, EML, Draft, Service Binding

Routing

manifest의 routes/targets와 getRouter().navTo()를 연결하고, onInit과 attachPatternMatched의 실행 시점을 구분했습니다.

Gateway Architecture

Fiori 요청이 OData URI로 Gateway에 전달되고, ABAP DPC_EXT 로직이 JSON/XML 응답으로 반환되는 구조를 정리했습니다.

Code-to-Data

CDS, ADBC, AMDP를 통해 애플리케이션 서버 처리와 DB pushdown 처리의 차이를 비교했습니다.

CO Process Learning

CO는 표준원가, 실제원가, 배부, 분할, 차이 정산이 연결되는 구조로 정리했습니다. SAP CODE ERP 프로젝트에서는 이 흐름을 기준으로 조직 구조, 원가요소, SKF, 활동단가, Fiori 분석 화면을 설계했습니다.

Standard Cost CK11N · KEKO

표준원가 추정 결과를 월중 원가 비교의 기준값으로 사용

Internal Order AUFK · ACDOCA

임시 비용 집계 단위에서 발생한 비용을 추적하고 정산 대상으로 연결

Settlement KO88 · COBRB

오더 잔액을 정산 규칙에 따라 Cost Center 또는 Profit Center로 이동

SKF COEPR

인원수, 면적, 출하건수, 작업시간을 배부 기준 수량으로 관리

Allocation KSU5 · T811C/L

Cycle/Segment, 송신·수신 규칙, SKF 비율로 공통비를 배부

Profitability ACDOCA · PA Segment

매출, 표준원가, 배부원가를 수익성 분석 단위로 집계

CO Data Design Notes

01 Hybrid Splitting Model

제조 비용을 LABOR, MACH, OVRH 기준으로 분할하고, 동일 단위 비용은 작업 실적 수량으로 동적 안분했습니다.

02 Hierarchy Parser

Cost Center Group과 조직 계층을 parent-child 구조로 읽고, Queue/Stack 패턴으로 실행 대상 Cost Center 범위를 생성했습니다.

03 SKF Generation

EMP, SPACE, SHIP, WHRS를 핵심 통계주요지표로 압축해 HR, 물류, 공장 간접비 배부 기준으로 사용했습니다.

04 Allocation Master

본사 공통비, 물류 허브 비용, 플랜트 관리 비용을 단계별로 하향 배부하는 Cycle/Segment 구조를 설계했습니다.

05 Actual Activity Price

분할된 실제 비용과 실제 조업도 수량을 연결해 활동단가를 산출하고, 표준원가와 실제원가 차이 분석으로 이어지게 했습니다.

06 Variance Settlement

표준 투입액과 실제 활동원가 차이를 비교해 가격차이와 배부차이를 구분하고 최종 원가 분석 기준으로 정리했습니다.

Mentoring Activity

멘토링 자료는 SAP 개발 레이어를 빠르게 연결하는 데 초점을 맞춰 정리했습니다. DB 접근, OData 서비스, Fiori 화면, RAP 앱 구현이 어떤 순서로 맞물리는지 주차별로 정리했습니다.

Week 1 ADBC · AMDP · RAP/OData

Open SQL과 Native SQL 접근 차이를 비교하고, ADBC 직접 DB 접근과 AMDP의 Code-to-Data 처리 방식을 정리했습니다.

Week 2 Fiori Master-Detail

Binding Context, OData Key Path, bindElement, Association, $expand를 중심으로 리스트와 상세 화면 흐름을 정리했습니다.

Week 3 Gateway · OData Request Flow

Fiori 요청이 OData URI를 통해 Gateway로 전달되고, DPC_EXT CRUD 메서드와 HANA 조회로 이어지는 흐름을 정리했습니다.

Week 3+ Metadata Extension · BDEF · EML

UI Annotation, Behavior Definition, Validation, Determination, Action, EML 기반 READ/MODIFY ENTITIES 흐름을 정리했습니다.

Connected Project

SAP CODE에서 학습한 개발 스택과 CO 프로세스는 EverNiture ERP 구축 프로젝트로 이어졌습니다. 프로젝트에서는 PM으로 일정과 산출물 흐름을 관리하고, CO 담당으로 표준원가, 실제 활동단가, SKF, 배부, 수익성 분석 화면을 설계했습니다.
