TON Aggregator & Arbitrage Bot

TON 생태계의 DEX 가격 비교, Telegram Mini App 접근성, 차익거래 실행 흐름을 하나의 서비스로 연결하는 아이디어를 기획했습니다.

Project Overview

STON.fi, DeDust 등 TON DEX 간 가격과 유동성 정보를 비교하고, 유리한 swap 경로와 arbitrage 기회를 Telegram 안에서 확인할 수 있는 봇/미니앱을 제안했습니다. 단순한 가격표가 아니라 수수료, 슬리피지, 유동성, 실행 가능성을 함께 계산하는 구조로 기획했습니다.

TON DeFi DEX Aggregator Arbitrage Bot Telegram Mini App

Problem Definition

  • TON DEX별 wallet interface, contract 호출 방식, 데이터 제공 방식이 서로 달라 가격 비교가 불편함
  • Telegram Mini App 안에서 즉시 사용할 수 있는 DeFi 서비스와 arbitrage 실행 봇이 부족함
  • 어디서 swap하는 것이 유리한지 직관적으로 안내하는 도구가 부족함
  • DEX 간 가격 차이가 발생해도 자동화된 실행 주체가 부족해 시장 균형화가 늦어질 수 있음
  • LP APR, staking 수익률, liquidity provide 조건을 한 화면에서 비교하기 어려움

Research Scope

초기 리서치에서는 STON.fi, DeDust, Megaton, Tegro, Optus, Flex 등 TON 기반 DEX와 관련 서비스를 비교했습니다. AMM 방식, order book 방식, aggregator 가능성, 실제 운영 여부, 유동성 규모, swap fee, oracle 연동 가능성을 검토했습니다.

DEX Liquidity 실질적으로 활용 가능한 DEX와 풀 유동성, 거래량, TVL을 우선 확인
Oracle Pyth, Redstone 등 TON 연동 가능 oracle을 후보로 검토
Interface DEX별 contract 호출 방식과 account/transaction type 차이를 통합할 수 있는지 검토

Core Idea

Aggregator

  • DEX별 swap 가격, pool 상태, fee, slippage 정보를 주기적으로 수집
  • 사용자가 입력한 pair와 수량 기준으로 최적 swap 경로 추천
  • LP 제공, staking, DeFi APR 정보를 함께 비교

Arbitrage Bot

  • DEX 간 가격 차이를 감지하고 순수익 기준으로 기회 필터링
  • Telegram 알림으로 실행 가능성을 사용자에게 전달
  • 실행 직전 데이터가 달라지면 transaction을 취소하는 안전장치 고려

Profit Logic

arbitrage는 가격 차이만으로 판단하지 않고 gas fee, swap fee, slippage, 가격 변동 리스크를 함께 계산해야 한다고 보았습니다. 최소 거래 규모와 실행 조건은 순수익 기준으로 결정합니다.

Net Profit = [(Sell Price - Buy Price) x Volume] - (Gas Fees + Swap Fees + Slippage Cost)
  • TON gas fee는 평균적으로 낮지만, 거래 수익이 작으면 swap fee와 slippage가 수익을 잠식할 수 있음
  • STON.fi swap fee 0.3%, slippage 0.1% 가정으로 손익분기점을 검토
  • 가격 차이가 threshold 이상일 때만 알림과 실행 후보를 생성하는 방식 제안

Architecture

Telegram Mini App

  • swap 비교, arbitrage 알림, educational guide를 제공하는 사용자 접점
  • TonKeeper, Tonhub, MyTonWallet 등 지갑 연결 흐름 고려

Off-chain Backend

  • DEX API와 oracle에서 가격, liquidity, pool 상태를 수집
  • PostgreSQL과 Redis 기반 저장/캐시, arbitrage detection scheduler 구성

TON Smart Contract

  • Arb Router Contract에서 multi-swap과 on-chain 실행 담당
  • signed transaction을 통해 사용자 지갑과 연결하고 최종 수익을 사용자 wallet으로 반환

Analytics

  • Tonapi, Ton Center, TonScan, TON Dune Analytics로 block/tx와 swap/liquidity 데이터 확인
  • 실행 전후 가격 차이와 pool 상태를 검증하는 데이터 파이프라인 구상

Implementation Challenges

  • TON 생태계의 낮은 TVL과 유동성은 arbitrage 수익성을 제한할 수 있음
  • DEX별 표준 인터페이스가 부족해 adapter layer가 필요함
  • 가격 오라클과 DEX API 데이터의 지연, 불일치, 신뢰성을 검증해야 함
  • front-running, stale price execution, contract 취약점 등 보안 리스크 관리가 필요함
  • 초보자가 Telegram 안에서 이해할 수 있는 UX/UI로 단순화해야 함

Expected Impact

  • DEX 간 가격 차이를 빠르게 해소해 TON 생태계의 가격 효율성을 높임
  • Telegram 안에서 DeFi 접근성을 높여 신규 사용자 진입 장벽을 낮춤
  • DEX 간 투명한 경쟁을 유도하고 liquidity provider의 선택 효율을 높임
  • 향후 Pro Version에서는 지속적인 arbitrage opportunity 탐지와 자동화 기능 확장 가능
